fury.js

Four methods to freedom.

fury.js is a mini-framework for JavaScript providing dead-simple, standalone:
  • Dependency Injection
  • Pub/sub-style Event handling and mediation
  • Collaboration with others with dependency on none: it loves to play with jQuery and Knockout
fury.js does not:
  • Require you to use any pattern, but I like MVC, MVVM (Presentation Model), and DI
fury.js owes its actual and conceptual existence to the Swiz Framework for Flex.

Sample Application

I've re-implemented TodoMVC by refactoring the Knockout.js version to use fury.js for event-driven development with dependency injection. Feel free to check it out online or as part of your git clone of fury.js's source.

Use

There's really only four methods to learn:

fury.inject( where, what, as, callback)

Inject a named dependency into another object as a given property name, then execute an optional callback. 
// Inject the "todoService" into this as "service"
fury.inject( this, "todoService", "service" );

fury.publish( eventName, data )

Send a publication to all subscribers, passing 'data' as a single argument. 
// Hey, someone clicked something!  Tell the world!
<input type="button" onclick="fury.publish( 'button.clicked', { greeting : 'Hi!' } )" />

fury.subscribe( eventName, target, functionReferenceOrName )

State that when a message is published, you want a certain function to be invoked within a given object. 
// When someone clicks something, alert the user
fury.subscribe( "button.clicked", this, function( data ) {
    alert( data.greeting );
));;

fury.register( { depedencyName : depedencyInstance } )

Register a map of dependency instances with fury.
(And, yes, they can be late-bound: you can register after you inject()!) 
// Set up our dependencies
fury.register({
    todoFormatter : new TodoFormatter(),
    todoPresentationModel : new TodoPresentationModel(),
    todoController : new TodoController(),
    todoService : new AmplifyTodoService()
})
